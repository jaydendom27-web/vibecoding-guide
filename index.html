과거 (Before)

개발자만 가능했던 일

코드 문법을 외우고, 오류를 고치고, 여러 파일을 연결해야 했습니다. 아이디어를 화면으로 옮기는 장벽이 매우 높았습니다.

현재 (Now)

비개발자도 가능한 일

AI가 코드 초안을 빠르게 만들어 줍니다. 우리는 "무엇을 왜 만들지"만 명확히 설명하면 됩니다.

바이브코딩이란?

말 한 번으로 끝나는 마법이 아니라,
설명하고 확인하고 다듬는 작업입니다.

좋은 결과는 좋은 설명에서 시작해, 몇 번의 수정 요청을 통해 완성됩니다.

설명

3

4

수정

AI의 역할

HOW 어떻게 구현할까

HTML, CSS, JavaScript, 인터랙션 구현을 AI가 빠르게 도와줍니다.

구체적으로 말해주지 않아도 AI가 어느정도는 알아서 만들어 줍니다. 그러나, 사람의 상상력과 기획력에 따라 결과는 확연히 달라집니다.

업무에서 무엇을 할 수 있을까?

핵심은 거창한 시스템 개발이 아니라, 반복 설명·입력·정리·공유가 필요한 일을 작은 화면으로 바꾸는 것입니다.

업무 적용 관점

문서 하나를
“눌러보고 쓰는 화면”으로 바꿉니다.

공지문, 엑셀 체크리스트, 회의 메모, 보고서 초안처럼 정적인 자료를 사용자가 바로 이해하고 입력하고 확인할 수 있는 웹페이지나 간단한 앱으로 전환할 수 있습니다.

Level 1📌

공지·가이드 페이지

긴 안내문을 카드형 구조로 바꿔 핵심만 빠르게 보게 합니다.

예시 AI 사용 가이드, 교육 안내, 행사 신청 방법, 신규 제도 안내 페이지

간단 계산기·판정 도구

사용자가 값을 넣으면 결과와 판단 기준을 즉시 보여줍니다.

예시 품질 이슈 중대성 판정, 교육 대상 우선순위 계산, 비용·일정 간이 산정
Level 2📝

입력폼·체크리스트

받아야 할 정보를 빠짐없이 입력하게 하고, 누락을 줄입니다.

예시 회의 안건 수집, 현장 점검 체크리스트, AI Needs 접수 화면
Level 3📊

현황판·브리핑 화면

여러 정보를 한 화면에 모아 경영층·팀장이 바로 보게 합니다.

예시 주간 업무 현황, 프로젝트 이슈 보드, 대표이사 모바일 브리핑
✓권장 개인·팀 단위의 설명, 정리, 입력, 간단 계산, 시연용 프로토타입부터 시작합니다.
!주의 로그인, 권한, 개인정보, 회사 핵심 데이터 저장이 필요한 앱은 정식 IT·보안 절차와 함께 검토합니다.
HHTML 권장 AI가 React로 만들어주는 경우가 있지만, 사내 시스템에서 바로 활용하려면 HTML/CSS/JavaScript 기반의 단일 HTML 파일로 요청하는 것이 안전합니다.
RReact란? 화면을 작은 부품처럼 나눠 만드는 웹 개발 라이브러리로, 별도 빌드·배포 환경이 필요한 경우가 많습니다.

아래 5가지만 지켜도 실습 결과가 훨씬 좋아집니다.

1

한 문장 정의

4

작게 쪼개기

한 번에 크게 말하지 말고, 뼈대부터 만든 뒤 살을 붙입니다.

5

결과 보고 수정

결과물을 본 뒤 "이 부분만 바꿔줘" 하고 반복 요청합니다.

프롬프트 입력 전, Copilot 도구에서 Pages를 먼저 설정해주세요 (프롬프트 입력창 왼쪽 아래 '+' 클릭 후 '더보기 > 페이지' 클릭)

Copilot 입력창에서 바로 프롬프트를 넣기 전에 도구 메뉴에서 Pages를 선택하면 결과 화면을 더 쉽게 확인하고 다듬을 수 있습니다.
결과를 보다가 제목, 버튼명, 설명 문구처럼 간단한 텍스트만 고치고 싶을 때는 Pages의 코드 화면에서 해당 문구를 찾아 직접 수정해도 됩니다.

미리보기가 켜지지 않는 경우, HTML 파일로 저장해서 확인하면 됩니다.

회사 보안 정책이나 Copilot Pages 설정에 따라 코드 미리보기가 활성화되지 않을 수 있습니다. 이때는 생성된 코드를 HTML 파일로 저장해 브라우저에서 직접 열어보면 됩니다.

  1. Copilot Pages 오른쪽 코드 화면에서 전체 HTML 코드를 선택합니다. Ctrl + A를 누른 뒤 Ctrl + C로 복사합니다.
  2. Windows 검색창에서 메모장을 검색해 실행합니다.
  3. 빈 메모장 화면에 복사한 코드를 Ctrl + V로 붙여 넣습니다.
  4. 상단 메뉴에서 파일 → 다른 이름으로 저장을 선택합니다.
  5. 파일 이름은 예를 들어 company-survival-test.html처럼 마지막 확장자를 반드시 .html로 입력합니다.
  6. 파일 형식은 모든 파일(*.*)로 변경하고, 인코딩은 UTF-8로 선택한 뒤 저장합니다.
  7. 저장된 HTML 파일을 더블클릭하면 Edge 또는 Chrome 브라우저에서 결과 화면을 확인할 수 있습니다.
주의: 파일 형식을 “텍스트 문서”로 두면 .html.txt로 저장될 수 있습니다. 파일명이 .html로 끝나는지 꼭 확인해주세요.

🧭 회사 생존 유형 테스트

회사에서 나는 어떤 방식으로 살아남는 사람일까? 가벼운 질문 몇 개로 알아보는 밈 테스트 페이지를 만듭니다. 아래 단계는 그대로 복사해 넣을 수 있고, Level 1 → Level 2 → Level 3로 갈수록 결과 차이가 눈에 보이게 구성했습니다.

LEVEL1

테스트 소개 페이지 만들기

🎯 목표
아직 질문 기능은 없음. 제목 · 짧은 소개 · 시작 버튼 · 4개 유형 미리보기 카드가 보이는 깔끔한 정적 소개 페이지.
💬 입력 프롬프트
회사 생존 유형 테스트 페이지를 HTML로 예쁘게 만들어줘. (미리보기 필요) 맨 위에는 제목이 크게 보이고, 바로 아래에 이 테스트가 어떤 건지 한두 줄짜리 짧은 소개가 있었으면 좋겠어. 그 아래에 "테스트 시작" 버튼을 크고 눈에 띄게 넣어줘. 아직 질문 기능은 넣지 말고, 어떤 결과 유형이 나오는지 미리 보여주는 소개 카드 4개를 가로로 깔끔하게 보여줘. 유형은 아래 4가지야. - 조용한 생존러 - 눈치 만렙 조율러 - 일머리 직진러 - 아이디어 생존러 각 카드에는 유형 이름이랑 한 줄 소개만 가볍게 들어가면 돼. 분위기는 요즘 밈 테스트 앱처럼 세련되고 가볍게 해줘. 배경은 흰색, 포인트는 네이비(#002C5F), 보조색은 바이올렛(#7C63D7) 정도로 쓰면 좋겠어. 모바일에서도 예쁘게 보이게 해줘.
✨ 기대 결과 / 다음 요청
테스트를 시작하기 전의 감성적인 소개 화면이 완성됨. 어떤 유형들이 있는지만 살짝 맛보는 단계.

👉 다음: "이제 시작 버튼을 누르면 실제 질문이 넘어가고 결과가 나오게 해보자."
LEVEL2

질문에 답하면 결과가 나오는 테스트 만들기

🎯 목표
시작 버튼 → 질문 한 개씩 → 보기 선택 → 다음 질문 → 마지막에 4개 유형 중 하나 결과 → 다시 해보기까지 되는 실제 인터랙티브 테스트.
💬 입력 프롬프트
방금 만든 회사 생존 유형 테스트 페이지를 이어서 수정해줘. 이제 "테스트 시작" 버튼을 누르면 실제 테스트가 진행되게 해줘. 질문은 한 번에 하나씩만 보이게 하고, 보기를 누르면 다음 질문으로 자연스럽게 넘어가게 해줘. 마지막 질문이 끝나면 결과 화면이 나오게 해줘. 질문은 회사에서 일할 때 내 스타일을 알아보는 느낌으로 5개 정도 만들어줘. 예를 들면 이런 주제들이야. - 회의에서 나는 어떤 편인지 - 급한 일이 생겼을 때 어떻게 움직이는지 - 보고할 때 나는 어떤 스타일인지 - 협업할 때 내 역할은 무엇인지 - 퇴근 직전 나는 어떤 모드인지 질문마다 보기는 4개씩 넣어주고, 보기를 고른 값에 따라 결과 유형이 정해지면 돼. 결과는 조용한 생존러, 눈치 만렙 조율러, 일머리 직진러, 아이디어 생존러 중 하나가 나오게 해줘. 결과 화면에는 내 유형 이름이 크게 보이고, 짧은 한 줄 설명이랑 특징 3가지 정도를 재밌게 적어줘. 그 아래에 "다시 해보기" 버튼도 넣어줘. 전체 분위기는 지금 만든 첫 화면 톤을 유지하면서, 요즘 밈 테스트 앱처럼 세련되고 가볍게 해줘. 배경은 흰색, 포인트는 네이비(#002C5F), 보조색은 바이올렛(#7C63D7) 정도로 쓰면 좋겠어. 모바일에서도 예쁘게 보이게 해줘.

추가 작업 · Level 3 심화

결과 화면을 밈 테스트처럼 다듬고 공유 요소 넣기

기능을 복잡하게 늘리지 말고, 완성된 테스트를 "공유하고 싶을 정도로 예쁘게" 다듬는 단계예요. 아래 프롬프트를 순서대로 하나씩 적용하면 됩니다. 3번은 선택이에요.

1
유형별 개성 살리기

네 가지 유형이 서로 분위기가 달라 보이도록 결과 화면에 색과 아이콘 포인트를 다르게 적용합니다.

결과 화면에 유형별 개성을 좀 더 살려줘. 네 가지 유형(조용한 생존러, 눈치 만렙 조율러, 일머리 직진러, 아이디어 생존러)이 서로 분위기가 달라 보이도록, 유형마다 대표 색상이랑 대표 아이콘(또는 이모지)을 조금씩 다르게 정해줘. 결과 카드 배경이나 포인트 색도 내가 나온 유형에 따라 자연스럽게 바뀌게 해줘. 기능은 건드리지 말고, 결과 화면 비주얼만 유형별로 개성 있게 다듬어줘. 전체 디자인 톤은 지금까지 만든 화면과 자연스럽게 이어지게 해줘. 모바일에서도 예쁘게 보이게 해줘.
2
공유하고 싶은 결과 카드 + 버튼 추가

결과 카드를 캡처해서 친구한테 보내고 싶을 정도로 다듬고, "다시 테스트하기" · "결과 문구 복사하기" 버튼을 붙입니다.

결과 화면을 공유하고 싶을 정도로 밈스럽고 귀엽게 다듬어줘. 결과 카드는 캡처해서 친구한테 보내고 싶을 만큼 깔끔하고 감각적으로 만들어줘. 그리고 결과 카드 아래에 버튼 2개를 넣어줘. - "다시 테스트하기" — 처음 소개 화면으로 돌아가거나 질문부터 다시 시작 - "결과 문구 복사하기" — 내 유형명이랑 설명이 클립보드에 복사되게 기능은 그대로 두고, 결과 화면만 더 밈 테스트답게 감성을 끌어올려줘. 전체 디자인 톤은 지금까지 만든 화면과 자연스럽게 이어지게 해줘. 모바일에서도 예쁘게 보여야 해.
3
(선택) 전체 디자인 수준 한 단계 끌어올리기

기능은 건드리지 않고, 폰트 위계·여백·색감·그림자 같은 디테일만 손봐서 "요즘 잘 만든 서비스" 느낌으로 마무리합니다.

지금 만든 회사 생존 유형 테스트를 전반적으로 더 고급스럽고 트렌디한 비주얼로 다듬어줘. 폰트 위계, 여백 간격, 색감 조화, 카드 모서리, 버튼 모양, 그림자 깊이까지 요즘 잘 디자인된 웹 서비스 느낌이 나게 디테일을 전반적으로 올려줘. 기능은 건드리지 말고, 보이는 수준만 한 단계 위로 끌어올리는 느낌으로 해줘. 전체 디자인 톤은 지금까지 만든 화면과 자연스럽게 이어지게 해줘. 모바일에서 봤을 때도 여백과 정렬이 예쁘게 잡혀 있어야 해.
