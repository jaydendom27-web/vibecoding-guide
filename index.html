과거 (Before)

개발자만 가능했던 일

코드 문법을 외우고, 오류를 고치고, 여러 파일을 연결해야 했습니다. 아이디어를 화면으로 옮기는 장벽이 매우 높았습니다.

현재 (Now)

비개발자도 가능한 일

AI가 코드 초안을 빠르게 만들어 줍니다. 우리는 "무엇을 왜 만들지"만 명확히 설명하면 됩니다.

바이브코딩이란?

말 한 번으로 끝나는 마법이 아니라,
설명하고 확인하고 다듬는 작업입니다.

좋은 결과는 좋은 설명에서 시작해, 몇 번의 수정 요청을 통해 완성됩니다.

1

설명

누가, 왜, 어떤 화면이 필요한지 사람의 언어로 전합니다.

2

생성

AI가 HTML·CSS·JS 초안을 빠르게 만들어냅니다.

3

확인

결과물을 실제로 보며 의도와 맞는지 체크합니다.

4

수정

아쉬운 점을 구체적으로 다시 요청하며 완성도를 올립니다.

준비확인 방식 이해

미리 보기는 1차 확인, 정확한 동작은 HTML 저장 후 실행합니다.

Level 1소개 화면

정적 페이지로 구조와 분위기를 잡습니다.

Level 2반응 기능

선택지·진행·결과 화면이 실제로 작동합니다.

Level 3디자인 심화

유형별 결과 카드와 공유 요소를 다듬습니다.

▶
Copilot ‘미리 보기’는 1차 확인용입니다

HTML 코드가 생성되면 답변 상단 오른쪽의 표시 옵션 아이콘(슬라이더 모양)을 누른 뒤 미리 보기를 선택하세요. 화면 구성과 분위기는 빠르게 볼 수 있지만, 버튼 클릭·질문 전환·복사 기능 등은 실제 브라우저 실행 결과와 다를 수 있습니다.

확인 순서: 미리 보기 → HTML 파일 저장 실행
  1. Copilot에 실습 프롬프트를 입력합니다.
  2. HTML 결과가 생성되면 답변 상단 오른쪽의 표시 옵션 아이콘을 클릭하고 미리 보기를 선택합니다.
  3. 미리 보기에서는 전체 레이아웃, 색감, 카드 배치, 결과 화면 분위기를 먼저 확인합니다.
  4. 정확한 버튼 동작과 화면 전환을 확인하려면 생성된 HTML 코드 전체를 복사합니다.
  5. Windows 메모장을 열어 코드를 붙여넣고, 다른 이름으로 저장을 선택합니다.
  6. 파일 형식은 모든 파일, 파일명은 예를 들어 company-test.html, 인코딩은 UTF-8로 저장합니다.
  7. 저장한 HTML 파일을 브라우저에서 열어 시작 버튼, 질문 선택, 결과 산출, 다시 해보기, 복사 기능이 정확히 작동하는지 확인합니다.
  8. 아쉬운 부분은 Copilot에 “방금 결과에서 ○○만 수정해줘”처럼 이어서 요청합니다.
진행 팁: Level 1은 미리 보기만으로도 화면 차이를 쉽게 확인할 수 있습니다. Level 2부터는 질문 전환과 결과 산출이 들어가므로 HTML 파일로 저장해 실행하면서 확인하는 것이 좋습니다.
Level 1 정적 소개 페이지

제목, 소개 문구, 시작 버튼, 네 가지 유형 카드가 보이는 첫 화면을 만듭니다.

결과물 차이: 아직 질문은 진행되지 않습니다. 미리 보기에서는 화면 구조와 분위기를 빠르게 확인합니다.
Level 2 실제 테스트 앱

시작 버튼, 질문 진행, 선택지 클릭, 결과 산출, 다시 해보기 기능이 붙습니다.

결과물 차이: 직접 답을 고르면 네 가지 유형 중 하나가 나옵니다. 정확한 질문 전환과 결과 산출은 HTML 파일로 저장해 확인합니다.
Level 3 공유하고 싶은 결과 화면

유형별 색상·아이콘·결과 카드·복사 버튼을 다듬어 완성도를 높입니다.

결과물 차이: 단순 테스트에서 “보여주고 싶은 밈 테스트” 느낌으로 바뀝니다. 복사·재시작 버튼은 브라우저 실행 상태에서 확인합니다.

🧭 회사 생존 유형 테스트

회사에서 나는 어떤 방식으로 살아남는 사람일까? 가벼운 질문 몇 개로 알아보는 밈 테스트 페이지를 만듭니다. Level 1 → Level 2 → Level 3로 갈수록 결과물이 어떻게 달라지는지 미리 보기로 먼저 확인하고, 실제 동작은 HTML 파일로 저장해 실행하면서 확인합니다.

LEVEL1

테스트 소개 페이지 만들기

🎯 목표
아직 질문 기능은 없음. 제목 · 짧은 소개 · 시작 버튼 · 4개 유형 소개 카드가 보이는 깔끔한 정적 소개 페이지.
💬 입력 프롬프트
회사 생존 유형 테스트 페이지를 HTML로 예쁘게 만들어줘. HTML, CSS, JavaScript를 하나의 파일 안에 넣은 완성형 단일 HTML 코드로 제공해줘. 맨 위에는 제목이 크게 보이고, 바로 아래에 이 테스트가 어떤 건지 한두 줄짜리 짧은 소개가 있었으면 좋겠어. 그 아래에 "테스트 시작" 버튼을 크고 눈에 띄게 넣어줘. 아직 질문 기능은 넣지 말고, 어떤 결과 유형이 나오는지 보여주는 소개 카드 4개를 가로로 깔끔하게 보여줘. 유형은 아래 4가지야. - 조용한 생존러 - 눈치 만렙 조율러 - 일머리 직진러 - 아이디어 생존러 각 카드에는 유형 이름이랑 한 줄 소개만 가볍게 들어가면 돼. 분위기는 요즘 밈 테스트 앱처럼 세련되고 가볍게 해줘. 배경은 흰색, 포인트는 네이비(#002C5F), 보조색은 바이올렛(#7C63D7) 정도로 쓰면 좋겠어. 모바일에서도 예쁘게 보이게 해줘.
✨ 기대 결과 / 다음 요청
Level 1 결과물은 “보는 화면”입니다. 제목, 소개, 시작 버튼, 유형 카드만 보이고 아직 클릭해도 테스트는 진행되지 않습니다.

👉 다음: 버튼을 누르면 질문이 넘어가고 실제 결과가 나오도록 Level 2로 확장합니다.
LEVEL2

질문에 답하면 결과가 나오는 테스트 만들기

🎯 목표
시작 버튼 → 질문 한 개씩 → 보기 선택 → 다음 질문 → 마지막에 4개 유형 중 하나 결과 → 다시 해보기까지 되는 실제 인터랙티브 테스트.
💬 입력 프롬프트
방금 만든 회사 생존 유형 테스트 페이지를 이어서 수정해줘. 수정된 결과도 HTML, CSS, JavaScript가 하나의 파일 안에 들어간 완성형 단일 HTML 코드로 다시 제공해줘. 이제 "테스트 시작" 버튼을 누르면 실제 테스트가 진행되게 해줘. 질문은 한 번에 하나씩만 보이게 하고, 보기를 누르면 다음 질문으로 자연스럽게 넘어가게 해줘. 마지막 질문이 끝나면 결과 화면이 나오게 해줘. 질문은 회사에서 일할 때 내 스타일을 알아보는 느낌으로 5개 정도 만들어줘. 예를 들면 이런 주제들이야. - 회의에서 나는 어떤 편인지 - 급한 일이 생겼을 때 어떻게 움직이는지 - 보고할 때 나는 어떤 스타일인지 - 협업할 때 내 역할은 무엇인지 - 퇴근 직전 나는 어떤 모드인지 질문마다 보기는 4개씩 넣어주고, 보기를 고른 값에 따라 결과 유형이 정해지면 돼. 결과는 조용한 생존러, 눈치 만렙 조율러, 일머리 직진러, 아이디어 생존러 중 하나가 나오게 해줘. 결과 화면에는 내 유형 이름이 크게 보이고, 짧은 한 줄 설명이랑 특징 3가지 정도를 재밌게 적어줘. 그 아래에 "다시 해보기" 버튼도 넣어줘. 전체 분위기는 지금 만든 첫 화면 톤을 유지하면서, 요즘 밈 테스트 앱처럼 세련되고 가볍게 해줘. 배경은 흰색, 포인트는 네이비(#002C5F), 보조색은 바이올렛(#7C63D7) 정도로 쓰면 좋겠어. 모바일에서도 예쁘게 보이게 해줘.
✨ 기대 결과 / 다음 요청
Level 2 결과물은 “작동하는 테스트 앱”입니다. 시작 버튼, 질문 전환, 보기 선택, 결과 산출, 다시 해보기가 실제로 작동합니다.

👉 다음: 기능은 유지하고, 결과 화면을 공유하고 싶을 만큼 예쁘게 다듬는 Level 3로 확장합니다.
추가 작업 · Level 3 심화

결과 화면을 밈 테스트처럼 다듬고 공유 요소 넣기

Level 3는 기능을 새로 복잡하게 늘리는 단계가 아니라, Level 2에서 만든 테스트의 결과 화면을 “공유하고 싶을 정도로 예쁘게” 다듬는 단계입니다. 미리 보기에서는 Level 2와의 시각적 차이를 먼저 보고, 복사·재시작 같은 기능은 HTML 파일로 저장해 브라우저에서 확인합니다.

1
유형별 개성 살리기

네 가지 유형이 서로 분위기가 달라 보이도록 결과 화면에 색과 아이콘 포인트를 다르게 적용합니다.

결과 화면에 유형별 개성을 좀 더 살려줘. 수정된 결과도 HTML, CSS, JavaScript가 하나의 파일 안에 들어간 완성형 단일 HTML 코드로 다시 제공해줘. 네 가지 유형(조용한 생존러, 눈치 만렙 조율러, 일머리 직진러, 아이디어 생존러)이 서로 분위기가 달라 보이도록, 유형마다 대표 색상이랑 대표 아이콘(또는 이모지)을 조금씩 다르게 정해줘. 결과 카드 배경이나 포인트 색도 내가 나온 유형에 따라 자연스럽게 바뀌게 해줘. 기능은 건드리지 말고, 결과 화면 비주얼만 유형별로 개성 있게 다듬어줘. 전체 디자인 톤은 지금까지 만든 화면과 자연스럽게 이어지게 해줘. 모바일에서도 예쁘게 보이게 해줘.
2
공유하고 싶은 결과 카드 + 버튼 추가

결과 카드를 캡처해서 친구한테 보내고 싶을 정도로 다듬고, "다시 테스트하기" · "결과 문구 복사하기" 버튼을 붙입니다.

결과 화면을 공유하고 싶을 정도로 밈스럽고 귀엽게 다듬어줘. 수정된 결과도 HTML, CSS, JavaScript가 하나의 파일 안에 들어간 완성형 단일 HTML 코드로 다시 제공해줘. 결과 카드는 캡처해서 친구한테 보내고 싶을 만큼 깔끔하고 감각적으로 만들어줘. 그리고 결과 카드 아래에 버튼 2개를 넣어줘. - "다시 테스트하기" — 처음 소개 화면으로 돌아가거나 질문부터 다시 시작 - "결과 문구 복사하기" — 내 유형명이랑 설명이 클립보드에 복사되게 기능은 그대로 두고, 결과 화면만 더 밈 테스트답게 감성을 끌어올려줘. 전체 디자인 톤은 지금까지 만든 화면과 자연스럽게 이어지게 해줘. 모바일에서도 예쁘게 보여야 해.
3
(선택) 전체 디자인 수준 한 단계 끌어올리기

기능은 건드리지 않고, 폰트 위계·여백·색감·그림자 같은 디테일만 손봐서 "요즘 잘 만든 서비스" 느낌으로 마무리합니다.

지금 만든 회사 생존 유형 테스트를 전반적으로 더 고급스럽고 트렌디한 비주얼로 다듬어줘. 수정된 결과도 HTML, CSS, JavaScript가 하나의 파일 안에 들어간 완성형 단일 HTML 코드로 다시 제공해줘. 폰트 위계, 여백 간격, 색감 조화, 카드 모서리, 버튼 모양, 그림자 깊이까지 요즘 잘 디자인된 웹 서비스 느낌이 나게 디테일을 전반적으로 올려줘. 기능은 건드리지 말고, 보이는 수준만 한 단계 위로 끌어올리는 느낌으로 해줘. 전체 디자인 톤은 지금까지 만든 화면과 자연스럽게 이어지게 해줘. 모바일에서 봤을 때도 여백과 정렬이 예쁘게 잡혀 있어야 해.
